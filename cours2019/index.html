Data types

A data type defines the operations that can be done on the data, the meaning of the data, and the way values of that type can be stored.

Primitive data types

  • Boolean type
  • Numeric type (integer, floating point, biginteger)
  • Character (String) type

Enumerated data types

Set of values (e.g. CLUB, DIAMOND, HEART, SPADE)


		enum Suit {CLUB, DIAMOND, HEART, SPADE}; 
	    

		data Suit = Club | Diamond | Heart | Spade

Composite data types

  • Arrays, lists...
  • Records (struct, tuple, product type)
  • Classes (contains fields and methods)
  • Algebraic data types

Classes in Python


		class Human(object):
		  def __init__(self, name):
		    self.name = name
		  def says(self, msg):
		    print(self.name + " says " + msg)
	    

		me = Human("Tristan")
		Human.says(me, "hello") #Tristan says hello
		me.says("hello")        #Tristan says hello

Class inheritance in Java


		abstract class Animal { abstract String talk(); }
		class Cat extends Animal {
		  String talk() { return "Meow!"; }
		}
		class Dog extends Animal {
		  String talk() { return "Woof!"; }
		}
	    

		static void letsHear(final Animal a) {
		  System.out.println(a.talk());
		}	  
		static void main(String[] args) {
		  letsHear(new Cat());
		  letsHear(new Dog());
		}

Polymorphism in a nutshell

A single name to represent multiple different types

  • for any type
    • parametric/generic polymorphism (list length)
  • for a constrained set of types
    • ad hoc polymorphism or overloading (==)
    • subtype/inclusion polymorphism (see previous example)
